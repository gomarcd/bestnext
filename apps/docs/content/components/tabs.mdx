## Overview

Tabs segment closely related information while keeping people within a single view. Each tab should expose content that shares the same context and data set. Avoid mixing unrelated workflows within the same tab strip.

Use short, descriptive labels and avoid multi-line names. If tab content contains forms, preserve state when switching tabs to prevent data loss.

## UX Guidelines

- Order tabs by task frequency or process sequence.
- Keep tab labels in sentence case and avoid icons alone; pair icons with text if desired.
- Display a clear indicator for the active tab—color and underline combination works best.

## Accessibility details

The component follows the WAI-ARIA “Tabs with Automatic Activation” pattern. Content for inactive panels stays in the DOM but hidden. Ensure that interactive elements inside panels have unique IDs if referenced by aria attributes. Provide focus styles that are visible on both light and dark backgrounds.
