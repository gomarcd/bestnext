## Overview

Text fields support short responses like names, emails, and search queries. Always pair the input with a visible label to maintain context and ensure accessibility. Use helper text sparingly to clarify formatting expectations or consequences.

When validation fails, present specific error messages and keep helper text visible so people can compare the correct format with the issue.

## UX Guidelines

- Use sentence case labels and avoid punctuation unless the label is a question.
- Reserve placeholder text for examples, not essential instructions.
- Align grouped fields to create a predictable scan pattern.

## Accessibility details

The component exposes `aria-describedby` for helper and error content. When the error string changes, screen readers are notified automatically. Keep the helper and error text concise so it remains readable for assistive technology.
